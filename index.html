School_Management

Overview

  • This solution implements the School Management system. Presentation projects follow the School_Management.Presentation.* root namespace convention.
  • UI uses WPF and MVVM (CommunityToolkit.Mvvm). Data access uses EF Core with Npgsql.

Requirements

  • .NET 10 SDK
  • Visual Studio 2026 (recommended) or the dotnet CLI
  • PostgreSQL (for local development) or configured connection string

Quickstart (local)

  1. Clone repository:
    • git clone <repo-url>
  2. Restore packages:
    • Restore NuGet Packages in Visual Studio or dotnet restore
  3. Build:
    • Rebuild Solution or dotnet build
  4. Run:
    • Set the presentation WPF project as startup in Visual Studio and run, or dotnet run from that project's folder.

Project structure (high level)

  • Application/ — Application layer (use cases, services)
  • Core/ — Domain models and enums
  • Infrastructure/ — Repositories, EF Core, data access
  • Presentation/ — WPF UI projects; each presentation project should use School_Management.Presentation.<Feature> root namespace

Namespace & csproj conventions

  • Presentation projects should set these MSBuild properties in their .csproj:
    • __AssemblyName__: School_Management.Presentation.<Feature>
    • __RootNamespace__: School_Management.Presentation.<Feature>
  • Update source namespace declarations to match the root namespace. Example:
    • namespace New_Student_Management.ViewModels -> namespace School_Management.Presentation.New_Student_Management.ViewModels
  • A small helper script is included to rename namespaces across .cs and .xaml files: scripts/rename-namespaces.ps1.

Coding standards

  • Target .NET 10, C# default for that target.
  • Nullable enabled, ImplicitUsings enabled — keep them as-is in project files.
  • Use CommunityToolkit.Mvvm for observable/viewmodel patterns.
  • Prefer expression-bodied members, async/await for I/O, and clear exception handling. Use dotnet format or an EditorConfig placed at repo root.

Testing & CI

  • Add unit tests under a /tests folder aligned with projects.
  • CI should at minimum: restore, build, run unit tests, and run code formatting checks.

Branching & PRs

  • Feature branches: feature/<issue>-short-description
  • Bugfix branches: bugfix/<issue>-short-description
